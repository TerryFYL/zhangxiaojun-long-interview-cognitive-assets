# 长访谈/播客挖掘协议 v0.1

> 本协议用于后续处理凉子、罗永浩、张小珺、播客、课程及其他长素材，规范抽取认知精华的流程和质量控制。
>
> 核心定义：**认知 = 知识 + 认识**，既包含新信息（知识），也包含对现实的深入理解和认知转变（认识），两者缺一不可。

---

## 1. 输入素材要求

- **素材类型**：长访谈、播客、课程、专题讲座等，单集时长建议>30分钟。
- **格式**：优先提供高质量音频文件及自动识别（ASR）文本稿。
- **附加信息**：访谈参与者信息、时间戳信息、上下文描述应尽可能完整。
- **质量要求**：
  - 语音清晰，无明显杂音；
  - 自动转录需经过抽样人工复核，不保证全量误差率，但需重点段落确认准确率。

---

## 2. 处理阶段与流程

| 阶段                          | 输入                       | 输出                             | 说明                                        |
|-----------------------------|--------------------------|--------------------------------|-------------------------------------------|
| 1. 原始转录 (raw transcript)      | 音频 + ASR文本               | 经初步校对的完整原始转录文本         | 抽样审核ASR准确性，人工校正明显错误                    |
| 2. 节目卡 (episode card)           | 原始转录文本                 | 节目概要卡，包含访谈主题、关键词、结构概览 | 概括主题、访谈人物背景、重点时间戳、核心亮点                  |
| 3. 认知边界卡 (cognitive boundary cards) | 节目卡 + 深度阅读转录       | 认知边界卡（knowledge + recognition）  | 提炼新增知识点和认知修正，标注来源片段和不足、改进行动           |
| 4. 容器与提问分析 (container/question analysis) | 认知边界卡                  | 容器分析卡、提问结构分析卡             | 区分访谈内容矿源，分析访谈容器结构和提问力，评价人格魅力          |
| 5. 综合与洞察 (synthesis)          | 各阶段产物合集               | 主题、知识体系和行动建议综合报告          | 融汇认知成果，提炼成长建议，形成认知闭环                      |

## 4. 内容矿源、访谈容器、提问力与人格魅力区分方法

| 维度       | 定义与解析                                               | 举例说明                           |
|----------|-------------------------------------------------------|--------------------------------|
| 内容矿源    | 访谈实质内容的知识和现场经验来源，可为嘉宾观点、数据、案例等                      | 嘉宾讲述创业经验；引用市场统计数据           |
| 访谈容器    | 载体与形式，包括访谈结构、主持风格、话题切换方式                                  | 开放问答、故事叙述、专题辩论           |
| 提问力     | 主持人或提问者提问的深度与技巧，推动认知深挖或知识点铺陈的能力                          | 提出根本性问题、反问、追问细节           |
| 人格魅力    | 访谈中表现出的个性特征或风格，感染力和可信度，增强内容吸引力                              | 幽默、真诚、逻辑严密、共情反应           |

## 5. 对照 Terry 自身的认知缺口模型

| 缺口类型     | 含义说明                                                 | 关联访谈分析指标                                                    |
|------------|--------------------------------------------------------|-----------------------------------------------------------------|
| 知识缺口     | 新知识点未被掌握或缺乏更新                                     | 访谈中出现新词汇、概念或数据未被吸收                                   |
| 认识缺口     | 对现实、状况的理解或视角偏不足                                  | 认知边界卡中指出“认识偏差”“世界观不足”等                              |
| 经验缺口     | 缺少实际操作、实践或感受反馈的补充                                | 嘉宾经验分享不足，缺乏具体案例与细节                                    |
| 提问缺口     | 提问深度、精准度不够，导致信息挖掘有限                             | 访谈提问浅尝辄止，缺乏二次及反问能力                                    |
| 耐心缺口     | 对长素材的筛选、推敲不足，急于总结，缺乏连续深入跟进                     | 内容泛泛而谈，缺少细致剖析，影响认知积累                                 |

## 6. 质量门槛

- **来源完整性**：
  - 明确素材出处（节目名、集数、访谈人等）。
  - 明确时间戳对应片段，便于后续查证。
- **时间戳精准**：
  - 关注重点认知片段需精确到秒级时间戳。
- **ASR复核**：
  - 必须进行抽样人工校对，确保核心内容准确无误。
  - 禁止完全依赖自动转写生产结果。
- **总结要求**：
  - 避免空泛、泛泛而谈的无内容总结。
  - 强调至少包含“知识增长”和“认识转变”。
  - 指出不足、提出改进建议，具有指导意义。

## 7. Agent 的使用条件区分

| Agent 类型                  | 适用场景                                      | 注意事项                                   |
|--------------------------|------------------------------------------|----------------------------------------|
| 被访者/主持人人格 Agent         | 模拟嘉宾或主持人格回答，辅助理解、表达和延伸                 | 应保持人格特性真实，避免偏离访谈风格及事实          |
| 素材检索/分析工具 Agent        | 作为快速检索、自动标注、初步聚合知识点和问答建议                   | 仅为辅助工具，需人工二次审核，不得全权替代内容解读   |

**总结**：  
- 当需模拟嘉宾风格提供交互体验适合用人格 Agent；  
- 当需批量、结构化处理素材时，适合用分析工具 Agent；  
- 两者逻辑和目标不同，不能混用。

## 8. 下一轮处理新访谈/播客具体步骤

1. **准备阶段**  
   - 收集完整音频、基础资料，获取自动转录文本。  
   - 进行抽样人工核对ASR质量。  

2. **转录校对与标注**  
   - 完成时间戳准确的文本校对。  
   - 标注重点内容与采访者身份。  

3. **制作节目卡**  
   - 根据转录抽取摘要，制作访谈主题、关键词、参与者信息卡。  

4. **生成认知边界卡**  
   - 深度研读，全流程提纯知识点和认知转变，结合时间戳。  
   - 标明知识增量、认识修正、问题类型与后续行动方案。  

5. **容器与提问结构分析**  
   - 从访谈形式层面拆解信息来源、访谈载体、提问技巧、人设魅力。  
   - 记录典型示例与时间戳。  

6. **综合报告撰写**  
   - 对照 Terry 自身认知缺口模型，分析个人见解不足。  
   - 汇总可行动建议和认知成长路径。  

7. **复盘与优化**  
   - 反馈给下轮访谈准备，调优提问设计、采访重点及后续跟进策略。
