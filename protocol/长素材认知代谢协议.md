# 长素材认知代谢协议

> 适用范围：长访谈、长播客、长课程、长对谈、直播回放、人物深聊。

## 一句话

长素材不是先做摘要，而是先做认知代谢：把材料拆成 **知识增量 + 认识变化 + 自我对照 + 行动修正**。

这里的“认知”固定包含两层：

- **知识**：事实、机制、案例、行业经验、术语、具体做法。
- **认识**：理解方式、判断框架、价值排序、问题感、世界观、自我校准。

因此，每条高价值内容都必须回答：

1. 它新增了什么知识？
2. 它修正了什么认识？
3. 它会如何改变 Terry 下一次判断、提问、写作或行动？

## 固定产物

每批长素材最终至少生成这些文件：

- `material_processing_design.md`：本批素材的角色定义和处理边界。
- `cognitive_boundary_cards.md`：认知边界卡，核心产物。
- `interview_container_analysis.md`：访谈/播客容器能力分析。
- `question_depth_framework.md`：问题深度框架，用于比较提问力。
- `long_interview_mining_protocol.md`：本批之后可复用的挖掘协议。
- `index.html`：本批素材的 HTML 认知工作台，用于搜索、筛选和浏览卡片。
- `_cognitive_assets_manifest.json`：生成信息、模型、数量和路径。

如果前置结构化抽取已经完成，还应保留：

- `episodes/*.json`：单集卡。
- `episodes_md/*.md`：可读单集卡。
- `claims.jsonl`
- `mental_models.jsonl`
- `decision_rules.jsonl`
- `questions.jsonl`
- `writing_material.jsonl`

## 目录约定

推荐结构：

```text
data/<material-slug>/
  json/                     # 原始 transcript JSON，可选
  structured/
    episodes/*.json          # 必需：单集结构化卡
    episodes_md/*.md         # 可选
    claims.jsonl             # 可选但推荐
    mental_models.jsonl      # 可选但推荐
    decision_rules.jsonl     # 可选但推荐
    questions.jsonl          # 可选但推荐
    writing_material.jsonl   # 可选但推荐
    synthesis/               # 本协议产物输出目录
```

当前通用脚本要求至少存在：

```text
data/<material-slug>/structured/episodes/*.json
```

每个 episode JSON 推荐包含：

```json
{
  "episode_id": "素材ID",
  "bvid": "如果来自B站则可保留",
  "title": "标题",
  "core_question": "这一集真正围绕什么问题",
  "one_sentence_takeaway": "一句话结论",
  "terry_relevance": [],
  "key_claims": [],
  "mental_models": [],
  "decision_rules": [],
  "open_questions": [],
  "writing_material": [],
  "asr_caveats": []
}
```

## 通用命令

默认处理张小珺目录：

```bash
python3 tools/build_long_interview_cognitive_assets.py
```

处理新访谈或播客：

```bash
python3 tools/build_long_interview_cognitive_assets.py \
  --root data/<material-slug>/structured \
  --source-json-dir data/<material-slug>/json \
  --material-name "<节目或访谈名>" \
  --host-name "<主持人或节目名>" \
  --material-role "内容矿源 / 提问力样本 / 访谈容器样本 / 人格魅力样本 / 自我对照材料" \
  --comparison-notes "与张小珺的容器能力、凉子的深问题、罗永浩的人格魅力作对照" \
  --cards-per-episode 6
```

输出默认在：

```text
data/<material-slug>/structured/synthesis/
```

生成 HTML 工作台：

```bash
python3 tools/build_cognitive_html_viewer.py \
  --synthesis-dir data/<material-slug>/structured/synthesis \
  --title "<节目或访谈名>认知代谢工作台"
```

HTML 是首选阅读界面，Markdown 是可版本化的源文件。工作台应该支持：

- 搜索认知卡片
- 按不足类型筛选
- 按集数筛选
- 展开/收起卡片详情
- 切换查看处理设计、容器分析、问题深度框架和挖掘协议

## 认知边界卡

`cognitive_boundary_cards.md` 是核心产物。

每张卡必须包含：

- 标题
- 来源：素材 ID + timestamp
- 访谈内容
- 知识增量
- 认识变化
- Terry 可能的原默认判断
- 不足类型
- 可迁移模型
- 行动修正
- 复核状态

不足类型固定为：

- 知识不足
- 经验不足
- 模型不足
- 判断过粗
- 问题意识不足
- 耐心不足
- 世界观不足

不要新增临时分类。确实不合适时，先归到最接近的一类，再在正文解释。

## 四种素材角色

处理长素材时，必须先判定它主要是什么：

| 角色 | 关注点 | 典型对象 |
| --- | --- | --- |
| 内容矿源 | 嘉宾/讲者输出了哪些有价值知识和判断 | 强嘉宾长访谈、专家播客 |
| 访谈容器样本 | 主持人如何撑开长时间对话 | 张小珺式长访谈 |
| 提问力样本 | 问题如何把人带向更深远处 | 凉子式访谈 |
| 人格魅力样本 | 表达、叙事、节奏如何维持注意力 | 罗永浩式长谈 |

一个素材可以有多重角色，但不能混在一起分析。主持人能力、嘉宾知识、人格魅力、内容价值要分层处理。

## Agent 判断

不要默认把每个素材都做成 Agent。

适合做 Agent：

- 有稳定可复用的问题风格、判断框架或方法论。
- 用户未来会反复调用它来生成问题、评估材料或辅助判断。
- 素材足够多，能支撑行为模式蒸馏。
- 目标是“可交互能力”，不是一次性总结。

不适合做 Agent：

- 只是材料很长，但主持人/嘉宾方法论不稳定。
- 当前价值主要是内容矿源，而不是人格或提问方式。
- 用户明确不想蒸馏该对象。
- 更适合做 checklist、协议、认知边界卡或素材检索工具。

当前规则：

- 张小珺：不做主持人人格 Agent；作为长访谈容器样本和内容矿源处理。
- 凉子：可作为“深问题 / 远方问题 / 提问力 Agent”候选。
- 罗永浩：可作为人格魅力、叙事张力、长时间表达控制样本。

## 质量门槛

- 所有关键判断必须带素材 ID 和 timestamp。
- ASR 字幕必须保留复核提醒，不把未复核文本当最终事实。
- 不输出空泛总结，例如“AI 很重要”“组织很重要”。
- 不把 Terry 的可能默认判断写成确定事实，要写“可能”。
- 不把“认知”窄化为 mental model，必须同时看知识增量和认识变化。
- 不急于全量处理，先让 MVP 批次进入真实使用闭环。

## 下一批素材执行顺序

1. 抽取 transcript。
2. 生成 episode cards 和 JSONL evidence bank。
3. 运行 `tools/build_long_interview_cognitive_assets.py`。
4. 运行 `tools/build_cognitive_html_viewer.py` 生成 `index.html`。
5. 先在 HTML 工作台里筛选和浏览认知边界卡，标出真正击中的卡。
6. 被击中的卡进入 Wiki / 母题系统。
7. 根据素材角色决定是否进一步做 Agent、skill、checklist 或写作素材库。
